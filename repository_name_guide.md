## üìö General Structure

All repositories must follow the pattern:

```

```

- **type** ‚Üí main category of the repository (see list below).
- **detail1** ‚Üí language, technology, or main name.
- **detail2** ‚Üí purpose, topic, or context.

## üìù Best Practices
- Use **lowercase letters** and **hyphens** to separate words.
- Be **specific** in `detail2` to make searching easier.
- Avoid unclear abbreviations.
- Use **English** whenever possible for broader reach.
